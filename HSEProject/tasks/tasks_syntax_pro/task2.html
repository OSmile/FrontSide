Хотелось бы заметить, что существует такой термин: «Программирование под дурака». Означает, что пользователь может вводить значения вопреки всем сказанным условиям:
Сказано ввести число от нуля, до пяти – он вводит число 125. В данном случае, вся Ваша логика пойдёт не так.
Запрограммируем алгоритм решения квадратного уравнения.
a*x2 + b*x + c = 0;

  • Введи значения с клавиатуры: «a», «b», «c»
  • Проверь, если значение «а» равно 0 – то x = - c / b. Также проверь, что b не нуль, на нуль делить нельзя
  • Значение «а» не равно нулю – считаем дискриминант: b * b – 4 * a * c
  • Если Дискриминант отрицательный – решений нет;
  • Если дискриминант равен нулю - решение одно: x = - b / (2 * a)
  • Если дискриминант положительный: x1 = (-b + sqrt(D)) / (2 * a), x2 = (-b – sqrt(D)) / (2 * a)
  • Выведи результаты на экран
  • Подумай, можно ли оптимизировать этот алгоритм? Много условий – тоже плохо
