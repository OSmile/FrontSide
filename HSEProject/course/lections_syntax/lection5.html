Итак, в предыдущих лекциях были показаны механизмы, как вводить/выводить информацию;
Как хранить данные в переменных, их область видимости;
Были затронуты арифметические операции.
Также были даны практические задания для отработки данных навыков.
Ко всему к этому хотелось бы добавить следующее:
Правилом хорошего тона является не использовать в коде независимые значения констант. Другими словами, очень плохо писать нечто подобное: int a = b + 5;. Что в данном контексте несет собой число 5? Как это понять? Используй выражение: const int value = 5;, добавь к нему комментарий, или дай осмысленное название, чтобы тот, кто поддерживал твой код понимал, какую логику ты закладывал.
Комментарии. Как уже было сказано, существуют в коде однострочные комментарии: // Комментарий . Удобно для предоставления краткой информации о строке кода.
Почитай о комментариях тут: RAVESLI. Подробно описаны виды комментариев, правила их составления, а также их применимость в коде. Будь готов ознакомиться с инструментами и правилами документирования кода.
Для корректного форматированного вывода информации, ознакомься с KVODO . Манипуляторы с объяснениями и примерами кода.
Повтори информацию о типах данных, она скоро нам пригодится.
